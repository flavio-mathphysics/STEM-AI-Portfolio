STEM AI Evaluation Guide
Executive Summary

This project presents a structured framework for evaluating AI-generated responses in STEM disciplines, with emphasis on Mathematics and Physics.

The methodology is designed to assess response quality, reasoning accuracy, scientific validity, and compliance with task requirements. The evaluation process follows a systematic approach that prioritizes objectivity, consistency, transparency, and evidence-based decision-making.

The ultimate goal is to improve the reliability and educational value of AI-generated content through rigorous quality assessment.

Scope

This framework is applicable to:

Mathematical problem solving
Physics reasoning tasks
Quantitative analysis
Scientific explanations
Educational STEM content
AI-generated solutions and justifications
Error detection and correction
Quality assurance reviews
Evaluation Standards

Each response is evaluated according to the following quality dimensions.

Accuracy

The response must produce correct calculations, valid conclusions, and scientifically sound statements.

Reasoning Quality

The logical pathway used by the model should be coherent, complete, and free of invalid assumptions.

Methodological Correctness

Appropriate formulas, principles, and problem-solving methods must be applied.

Clarity

The response should communicate ideas clearly and present solutions in an understandable manner.

Completeness

All relevant aspects of the prompt should be addressed.

Evaluation Process
Step 1 — Prompt Assessment

Review the prompt to identify:

Subject domain
Required task
Expected outcome
Relevant constraints
Objective

Determine the criteria that define a successful response.

Step 2 — Response Analysis

Examine the AI-generated response and verify:

Correctness of calculations
Logical consistency
Scientific validity
Alignment with prompt requirements
Objective

Assess whether the response satisfies the task requirements.

Step 3 — Error Identification

When issues are detected, classify them according to standardized categories.

Common Error Categories
Calculation Error
Algebraic Error
Formula Error
Unit Conversion Error
Conceptual Error
Logical Reasoning Error
Interpretation Error
Objective

Identify the root cause of the failure.

Step 4 — Evidence-Based Evaluation

Document findings using objective reasoning.

The evaluation should:

Identify the error
Explain why it occurred
Reference the correct principle
Provide supporting evidence
Objective

Ensure transparency and reproducibility.

Step 5 — Corrective Solution

Provide the correct solution using appropriate methods and reasoning.

Requirements:

Show calculations when relevant
Apply correct formulas
Verify units
State the final answer clearly
Objective

Demonstrate the expected response quality.

Step 6 — Quality Control Review

Before finalizing the evaluation:

Verify calculations
Confirm formula selection
Check unit consistency
Review explanatory clarity
Ensure completeness
Objective

Maintain evaluation reliability and consistency.

Guiding Principles
Objectivity

Evaluations are based on evidence rather than personal opinion.

Consistency

Equivalent errors should receive equivalent classifications and justifications.

Transparency

All conclusions must be supported by explicit reasoning.

Accuracy

Corrections must be mathematically and scientifically valid.

Continuous Improvement

Evaluation standards should evolve as new tasks, domains, and AI capabilities emerge.
